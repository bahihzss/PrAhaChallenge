# 課題１

* 追加、削除の処理が複雑になる
  * すべてアプリケーション側で `split` → `join` を行う必要がある
* 削除だろうと追加だろうと UPDATE になるので、クエリだけで何やってるかわかりづらい
* 検索も正規表現などを使う必要が出てくる
* 文字列だとどんな値でも入れれてしまう
* タグをネストしたい、既にあるタグを補完で出したいなど、 tags テーブルが必要になるような要件が出た場合に、マイグレーションが大変
* 文字数制限を超えたらタグを追加できなくなるが、タグ名の長さが固定でなければ何個まで追加できるか予測できない
